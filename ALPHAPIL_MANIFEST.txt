ALPHAPIL TECHNICAL MANIFEST (V0.4.0)
====================================
Guide for AI Agents to write professional AlphaPIL templates.

1. CORE ENGINE ARCHITECTURE
---------------------------
- PARSING: Recursive, inside-out. Nested functions resolve FIRST.
- AA (Anti-Aliasing): 1 (Off) to 8 (High). Smooths edges via supersampling.
- SCALE: Output resolution multiplier (e.g., scale=2 for Retina/HiDPI). Independent of AA.
  - Use `scale` for resolution, `aa` for smoothness. scale=3;aa=2 is ideal for most use cases.
- QUALITY PIPELINE: Lanczos downsample → UnsharpMask sharpening → DPI metadata embedding.
- BOX MODEL: $startContainer[x;y;w;h;padding=0] defines a local context with optional internal padding.
- CLIPPING: $startClip[x;y;w;h;radius=0] prevents any drawing outside the region.
- AUTO-FONT: Automatically discovers fonts on the user's device (Win/Mac/Linux) by name.
- ERROR HANDLING: `strict=false` captures errors in $getErrors[] without crashing.

2. COORDINATE MASTERY
---------------------
- KEYWORDS: left, right, top, bottom, center, middle (Context-aware: respect Padding).
- MATH: center+20, right-50, 100+10%.
- MIDPOINT: mid(p1,p2) or between(p1,p2).

3. COLORS & STYLING
-------------------
- ALPHA SYNTAX: color/alpha (e.g., white/20, #ff9900/40, rgb(255,0,0)/60).
- ALTERNATIVE: color,alpha (e.g., #ffffff,10).

4. FUNCTION REFERENCE
---------------------
(Syntax: $func[param1;param2;...])
- AI Note: Prefer NAMED arguments for clarity (e.g., $drawRect[x=10;y=10;width=100;height=100]).

5. PROFESSIONAL BEST PRACTICES
------------------------------
1. GLASSMORPHISM: $blur[20;x;y;w;h] -> $drawRect[x;y;w;h;fill=white/10;outline=white/20;lw=1]
2. CLIPPING: Wrap charts or images in $startClip to ensure they never bleed out of their cards.
   - **Chart Label Tip**: When clipping charts, set the chart height to less than 100% (e.g. `100%-30`) to leave room for the bottom labels so they aren't clipped out.
   - **Clipping Coordinate Context**: Because `$startClip` establishes a new local coordinate context, any drawing commands inside the clip block (like `$drawImage`) should be positioned relative to this local box (e.g. using `x=0;y=0` to align perfectly with the clip origin) rather than using absolute coordinates. Using absolute coordinates inside a clip shifts them a second time, pushing images off-center or completely out of their boundaries.
3. PADDING: Use $startContainer[...;padding=20] so you can just use 'left' and 'top' for content.
4. TYPOGRAPHY: Use 'tracking' to give headers a premium feel.
5. QUALITY: Use `scale=3;aa=2` instead of `aa=8` for best quality at smallest render cost.
   - scale=3 outputs at 3x resolution (e.g. 420x520 → 1260x1560).
   - aa=2 gives smooth edges without the massive memory cost of aa=8.
   - The engine automatically applies UnsharpMask sharpening after downsampling.

6. VISUAL COORDINATE PICKER (NEW)
---------------------------------
To easily inspect, find, and copy pixel coordinates on any generated canvas:
- COMMAND LINE INTERFACE:
  `alphapil-picker <path_to_image>`
- PYTHON PROGRAMMATIC API:
  `from alphapil import open_coordinate_picker`
  `open_coordinate_picker("<path_to_image>")`
- CANVAS ENGINE CONVENIENCE:
  `engine.open_coordinate_picker("<path_to_image>")`
This spins up a premium local HTML/JS Web server with crosshair reticles, real-time pixel scaling, single-click copy, and drag-and-drop support.
